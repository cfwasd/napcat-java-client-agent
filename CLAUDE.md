# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project Overview

NapCat Java SDK is a multi-module Maven project providing a Java Bot development framework based on NapCat (OneBot11 protocol), with integrated AI Agent capabilities.

## Build Commands

```bash
# Build all modules and install to local Maven repository
mvn install -DskipTests

# Build a single module (e.g., napcat-core)
mvn install -pl napcat-core -am -DskipTests

# Run the example bot application (entry point)
mvn spring-boot:run -pl napcat-admin

# Package the admin module as an executable JAR
mvn package -pl napcat-admin -am -DskipTests
```

## Module Architecture

- `napcat-core`: OneBot11 protocol implementation, communication adapters (HTTP/WS client/server), event routing, SQLite persistence (`DbManager`, `ScheduleStore`).
- `napcat-agent`: LLM Agent engine. Contains `NapCatAgent` (ReAct loop), `ToolRegistry`, session management, and the persistent memory subsystem (`MemoryStore`, `MemoryExtractor`, `DailyMemorySummarizer`).
- `napcat-llm-providers`: OpenAI-compatible, Anthropic Claude, and Ollama provider implementations. All implement `LlmProvider`.
- `napcat-spring-boot-starter`: Auto-configuration (`NapCatAutoConfiguration`) that wires everything together, including adapter selection based on `napcat.adapter.type`.
- `napcat-admin`: Example Spring Boot application. Add bot classes here under `com.napcat.admin.bot`. Main class: `com.napcat.admin.NapCatApplication`.

## Key Architecture Patterns

### Event Flow
1. `BotAdapter` receives raw OneBot11 events.
2. `MessageRouter` routes to `EventDispatcher`.
3. `EventDispatcher` matches handlers via `HandlerRegistry`.
4. Handler methods use annotations (`@OnGroupMessage`, `@OnPrivateMessage`, `@Command`) and receive typed events (`GroupMessageEvent`, `PrivateMessageEvent`).

### Agent ReAct Loop
- `NapCatAgent.chat()` initiates a session, injects system prompt + persistent memories, then enters `reactLoop()`.
- `reactLoop()` calls `llmProvider.chat()`; if the response has `toolCalls`, it invokes them via `ToolRegistry` and recurses up to `maxRounds`.
- After the final assistant response, `MemoryExtractor.extractIfNeeded()` runs asynchronously to extract structured memories from the conversation.

### Persistent Memory Subsystem (Two-Layer)
- `memories` table: Fragmented structured memories (fact / preference / topic) extracted by `MemoryExtractor` after conversations.
- `memory_summaries` table: Daily cumulative summaries generated by `DailyMemorySummarizer`.
- **Retrieval priority**: `retrieve()` checks `memory_summaries` first, then falls back to `memories`.
- **Injection timing**: `NapCatAgent` injects retrieved memories into the system prompt **only when `session.getHistory().isEmpty()`** (new session). It does NOT re-inject on subsequent turns in the same session.

### Scheduling
- `SchedulePoller` polls SQLite `schedules` table every 5 minutes and registers near-future tasks into `TimerWheel`.
- `TaskExecutor` handles three action types: `send_message`, `ai_generate`, `custom`.
- The daily memory summarizer is registered as a `custom` action task named `"每日记忆归纳"` with cron `0 0 1 * * ?`.

### Session Isolation
- Sessions are keyed by `(userId, groupId)` using `SessionKey`. Private chats use `groupId = 0`.
- `SessionManager` holds in-memory sessions with TTL and max history truncation.
- `SessionManager.clearExpired()` persists full session history to `memories` (type=`full_session`) before removal.

## Important Code Conventions

- **Annotations for bots**: `@Component` + `@OnGroupMessage` / `@OnPrivateMessage` / `@Command`. Return values are automatically sent as replies.
- **Tool registration**: Methods annotated with `@Tool` inside Spring `@Component` beans are auto-discovered by `ToolRegistry` in `NapCatAutoConfiguration`.
- **Role filtering**: Use `@RoleFilter(RoleFilter.Role.SUPERUSER)` for admin-only commands.
- **Database migrations**: `MigrationManager` in `NapCatAutoConfiguration` runs numbered migrations on startup. New schema changes should be registered there.
- **Configuration**: See `napcat-admin/src/main/resources/application.yml` for all available options. Key prefixes: `napcat.adapter`, `napcat.bot`, `napcat.llm`, `napcat.agent`, `napcat.memory`, `napcat.scheduler`.
